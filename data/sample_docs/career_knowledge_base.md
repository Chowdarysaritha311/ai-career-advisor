# AI/ML Career Roadmap — Knowledge Base

## Transitioning to Machine Learning Engineering

### Prerequisites
- Strong Python programming (NumPy, Pandas, Scikit-learn)
- Mathematics: Linear Algebra, Calculus, Probability & Statistics
- At least 1 deployed ML project in your portfolio
- Understanding of model lifecycle: training, evaluation, deployment

### Step-by-Step Transition Plan (6–12 Months)

**Months 1–2: Foundations**
- Complete Andrew Ng's Machine Learning Specialization on Coursera
- Practice Python data manipulation with Pandas daily
- Build 2 end-to-end projects on Kaggle

**Months 3–4: Deep Learning & Frameworks**
- Learn PyTorch or TensorFlow
- Study CNNs, RNNs, Transformers
- Implement papers from arXiv in code

**Months 5–6: MLOps & Production**
- Learn MLflow for experiment tracking
- Study Docker, Kubernetes basics
- Deploy a model using FastAPI + Docker

**Months 7–12: Specialisation & Job Search**
- Pick a niche: NLP, Computer Vision, or Tabular ML
- Contribute to open source (Hugging Face, PyTorch ecosystem)
- Build a portfolio of 3–5 polished GitHub projects
- Apply to 5 jobs per week and track progress

## Resume Tips for AI/ML Roles

### What Recruiters Look For
1. **Quantified impact**: "Improved model accuracy from 82% to 91%, reducing false positives by 34%"
2. **Tech stack specificity**: List exact tools — PyTorch 2.0, scikit-learn 1.4, FAISS
3. **End-to-end ownership**: Show you've taken a project from problem statement to production
4. **Published work**: GitHub repos, Kaggle notebooks, blog posts, papers

### Resume Structure for ML Engineers
- **Summary** (3 lines): Years of experience, specialisation, impact
- **Technical Skills**: Languages, Frameworks, Cloud, MLOps tools
- **Experience**: Role → Company → Dates → 3–4 bullet points with metrics
- **Projects**: Name → Tech stack → Link → 1 sentence of impact
- **Education**: Degree, Institution, Year (+ relevant certifications)

### Common Resume Mistakes
- Listing tools without showing application ("used TensorFlow" vs "trained ResNet-50 on custom 50K image dataset")
- No GitHub links or portfolio
- Generic descriptions without numbers
- Including irrelevant work experience without framing it for ML

## Interview Preparation Guide

### Technical Interview Topics

**Machine Learning Fundamentals**
- Bias-variance tradeoff
- Regularisation (L1 vs L2)
- Cross-validation strategies
- Gradient descent variants (SGD, Adam, RMSProp)
- Overfitting prevention techniques

**Deep Learning**
- Backpropagation mechanics
- Attention mechanisms and Transformers
- CNN architectures (ResNet, VGG, EfficientNet)
- Recurrent networks (LSTM, GRU)
- Batch normalisation, dropout, weight initialisation

**System Design (ML)**
- Designing a recommendation system at scale
- Building a real-time fraud detection pipeline
- Training infrastructure for large models
- A/B testing for model rollouts

**Coding Rounds**
- Implement k-means from scratch
- Write a logistic regression gradient update
- Sliding window / two-pointer for data problems
- SQL queries for business metrics

### Behavioural Questions (STAR Format)
- Describe a time your model failed in production — what did you do?
- How do you handle disagreements with stakeholders about model decisions?
- Tell me about a project where you had to learn a new technology quickly.
